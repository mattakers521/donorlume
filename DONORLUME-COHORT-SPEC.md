# DonorLume — Cohort Intelligence System Spec

*Feed this file to Claude Code along with the existing codebase. This spec adds the cohort intelligence layer on top of the existing Donor, DonorList, and scoring infrastructure.*

---

## Overview

The cohort system automatically classifies every donor into multiple overlapping cohorts the moment data is uploaded, then makes those cohorts filterable, scorable, and actionable across every module in the app. A donor can belong to many cohorts simultaneously (e.g., "Major Donor" + "Gala Attendee" + "Lapsed" + "Churn Risk").

---

## 1. Database Schema Additions

Add these models to the existing Prisma schema:

```prisma
// ─── COHORT SYSTEM ───

model CohortDefinition {
  id          String      @id @default(cuid())
  orgId       String
  name        String      // "Major Donors", "Gala 2024 Attendees", etc.
  slug        String      // "major-donors", "gala-2024-attendees"
  family      CohortFamily
  type        CohortType  // SYSTEM (auto-generated) or CUSTOM (user-created)
  description String?
  
  // For SYSTEM cohorts, this defines the auto-classification rule
  // Stored as JSON: { field: "totalGiven", operator: "gte", value: 10000 }
  // Or compound: { logic: "and", rules: [...] }
  rule        Json?
  
  // For tag-based cohorts (from CSV column values)
  sourceColumn String?    // e.g., "donor_type", "event", "segment"
  sourceValue  String?    // e.g., "Gala Attendee", "Corporate Sponsor"
  
  color       String?     // hex color for UI badges
  icon        String?     // lucide icon name
  sortOrder   Int         @default(0)
  isArchived  Boolean     @default(false)
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  org         Organization @relation(fields: [orgId], references: [id], onDelete: Cascade)
  members     DonorCohort[]

  @@unique([orgId, slug])
  @@index([orgId, family])
}

model DonorCohort {
  id                String    @id @default(cuid())
  donorId           String
  cohortDefinitionId String
  assignedAt        DateTime  @default(now())
  assignmentType    String    @default("auto") // "auto" | "manual" | "csv"
  confidence        Float?    // 0-1 for trajectory cohorts

  donor             Donor     @relation(fields: [donorId], references: [id], onDelete: Cascade)
  cohort            CohortDefinition @relation(fields: [cohortDefinitionId], references: [id], onDelete: Cascade)

  @@unique([donorId, cohortDefinitionId])
  @@index([cohortDefinitionId])
  @@index([donorId])
}

enum CohortFamily {
  GIVING_BEHAVIOR   // Major, mid-level, sustainer, first-time, lapsed, etc.
  ENGAGEMENT        // Gala attendees, volunteers, email engagers, etc.
  ENTITY_TYPE       // Individual, corporate, foundation, DAF, etc.
  TRAJECTORY        // Rising star, churn risk, upgrade candidate, etc.
  CUSTOM            // User-defined
}

enum CohortType {
  SYSTEM            // Auto-generated by rules
  CUSTOM            // Created by user
}
```

Also add to existing Donor model:
```prisma
model Donor {
  // ... existing fields ...
  cohorts     DonorCohort[]
}
```

And to Organization:
```prisma
model Organization {
  // ... existing fields ...
  cohortDefinitions CohortDefinition[]
}
```

## 2. System Cohort Definitions (Auto-Created Per Org)

When an organization is created (or first uploads donors), seed these system cohorts. Thresholds should be configurable in OrgSettings but these are the defaults:

### Giving Behavior Family

| Cohort | Rule | Color |
|--------|------|-------|
| Major Donors | totalGiven >= 10000 OR largestGift >= 5000 | #E8860C (amber) |
| Mid-Level Donors | totalGiven >= 1000 AND totalGiven < 10000 | #F5B731 (gold) |
| Small/Grassroots Donors | totalGiven > 0 AND totalGiven < 1000 | #6E6E73 (gray) |
| Recurring Sustainers | donorType contains "recurring" OR "sustainer" OR "monthly" OR totalGifts >= 6 AND frequency is regular | #34C759 (green) |
| First-Time Donors | totalGifts == 1 | #007AFF (blue) |
| One-Time Donors | totalGifts == 1 AND lastGiftDate > 12 months ago | #AEAEB2 (light gray) |
| Lapsed (LYBUNT) | gave in the prior fiscal year but not in current | #D44A1A (orange) |
| Lapsed (SYBUNT) | gave in some prior year but not current or last | #B91C1C (red) |
| Upgraders | average of last 2 gifts > average of prior gifts by 20%+ | #34C759 (green) |
| Downgraders | average of last 2 gifts < average of prior gifts by 20%+ | #FF3B30 (red) |
| Legacy/Planned Giving Prospects | tenure > 5 years AND totalGifts > 10 AND age indicators suggest 60+ (if available) | #AF52DE (purple) |

### Engagement Family (populated from CSV tags)

These are created dynamically when a CSV is uploaded and categorical columns are detected:
| Column Value Examples | Cohort Created |
|----------------------|----------------|
| "Gala", "Gala 2024", "Gala Attendee" | Gala Attendees |
| "Volunteer", "Vol" | Volunteers |
| "Board", "Board Member" | Board Members |
| "Event", "5K Run", "Auction" | Event Attendees (or specific event name) |
| "Email Subscriber" | Email Subscribers |
| "Peer Fundraiser", "P2P" | Peer-to-Peer Fundraisers |

### Entity Type Family

| Cohort | Rule |
|--------|------|
| Individual Donors | donorType matches "individual" or is blank/default |
| Corporate Donors | donorType matches "corporate", "company", "business", "sponsor" |
| Foundation / Grantmaker | donorType matches "foundation", "trust", "fund", "grantmaker" |
| Donor-Advised Fund | donorType matches "daf", "donor advised", "donor-advised" |

### Trajectory Family (computed, not tag-based)

| Cohort | Logic | Confidence |
|--------|-------|------------|
| Rising Stars | Small donor (<$1K total) but increasing gift frequency and/or amount over last 3+ gifts | 0.0-1.0 |
| Churn Risk | Active donor whose engagement signals are declining: longer gaps between gifts, decreasing amounts, or reduced frequency | 0.0-1.0 |
| Reactivation Ready | Lapsed but high reactivation score (70+) in the RFM engine | Score-based |
| Upgrade Candidates | Mid-level donors with giving patterns suggesting capacity for major gifts | 0.0-1.0 |
| Stewardship Priority | Recent gift significantly above their average — they stretched for you, acknowledge it | Automatic |

## 3. Auto-Classification Engine

### On CSV Upload (runs after scoring)

```typescript
// lib/cohort-engine.ts

interface ClassificationResult {
  donorId: string;
  cohortSlug: string;
  assignmentType: "auto" | "csv";
  confidence?: number;
}

export function classifyDonors(
  donors: ScoredDonor[],
  cohortDefinitions: CohortDefinition[],
  orgSettings: OrgSettings
): ClassificationResult[] {
  const results: ClassificationResult[] = [];

  for (const donor of donors) {
    // 1. Giving behavior cohorts (rule-based)
    for (const cohort of cohortDefinitions.filter(c => c.family === "GIVING_BEHAVIOR")) {
      if (evaluateRule(cohort.rule, donor)) {
        results.push({ donorId: donor.id, cohortSlug: cohort.slug, assignmentType: "auto" });
      }
    }

    // 2. Entity type cohorts (from donorType field)
    for (const cohort of cohortDefinitions.filter(c => c.family === "ENTITY_TYPE")) {
      if (matchesEntityType(cohort, donor.donorType)) {
        results.push({ donorId: donor.id, cohortSlug: cohort.slug, assignmentType: "auto" });
      }
    }

    // 3. Engagement cohorts (from CSV tag columns)
    // These are created during CSV parsing when categorical columns are detected
    for (const cohort of cohortDefinitions.filter(c => c.family === "ENGAGEMENT")) {
      if (donor._csvTags?.[cohort.sourceColumn] === cohort.sourceValue) {
        results.push({ donorId: donor.id, cohortSlug: cohort.slug, assignmentType: "csv" });
      }
    }

    // 4. Trajectory cohorts (computed from patterns)
    const trajectories = computeTrajectories(donor);
    for (const t of trajectories) {
      results.push({ 
        donorId: donor.id, 
        cohortSlug: t.slug, 
        assignmentType: "auto",
        confidence: t.confidence 
      });
    }
  }

  return results;
}
```

### CSV Column Detection for Engagement Cohorts

When parsing a CSV, detect categorical columns (columns where values repeat and aren't numeric/date):

```typescript
// lib/csv-cohort-detection.ts

export function detectCohortColumns(
  parsedRows: Record<string, string>[],
  headers: string[]
): DetectedCohortColumn[] {
  const results: DetectedCohortColumn[] = [];
  
  for (const header of headers) {
    // Skip columns already mapped to donor fields (name, email, dates, amounts)
    if (isMappedField(header)) continue;
    
    const values = parsedRows.map(r => r[header]).filter(Boolean);
    const unique = new Set(values);
    
    // Categorical: has repeated values, reasonable cardinality (2-50 unique values)
    // and values aren't all numbers or dates
    if (unique.size >= 2 && unique.size <= 50 && !allNumeric(values) && !allDates(values)) {
      results.push({
        column: header,
        uniqueValues: [...unique],
        sampleSize: values.length,
        suggestedCohortFamily: "ENGAGEMENT",
      });
    }
  }
  
  return results;
}
```

After detection, show the user a confirmation step: "We found these columns that look like donor segments. Which ones should become cohorts?"

## 4. API Routes

```
// Cohort definitions
GET    /api/cohorts                      — List all cohort definitions for the org
POST   /api/cohorts                      — Create custom cohort
PATCH  /api/cohorts/[id]                 — Update cohort (name, rules, color)
DELETE /api/cohorts/[id]                 — Delete custom cohort (not system ones)

// Cohort membership
GET    /api/cohorts/[id]/donors          — List donors in a specific cohort (paginated)
GET    /api/donors/[id]/cohorts          — List cohorts for a specific donor
POST   /api/donors/[id]/cohorts          — Manually add donor to a cohort
DELETE /api/donors/[id]/cohorts/[cohortId] — Remove donor from a cohort

// Classification
POST   /api/cohorts/classify             — Re-run auto-classification for the org
GET    /api/cohorts/detect-columns       — Detect cohort-eligible columns from latest upload

// Cohort analytics
GET    /api/cohorts/summary              — Count per cohort, total value, avg score
GET    /api/cohorts/[id]/insights        — Detailed stats for one cohort
```

## 5. UI Components

### 5.1 Cohort Filter Bar (appears on every data page)

A horizontal bar above the data table on /lapsed, /outreach donor selection, and /dashboard. Contains:
- Dropdown grouped by family: "Giving Behavior ▸", "Engagement ▸", "Entity Type ▸", "Trajectory ▸"
- Selected cohorts shown as removable pills
- "Clear All" link
- Active filter count badge

When a cohort filter is active, all data on the page respects it — scores, counts, KPIs, and table rows.

### 5.2 Cohort Badges on Donor Rows

Each donor row in the lapsed scoring table shows small colored badges for their cohorts. Max 3 visible + "+N more" overflow. Clicking a badge filters the table to that cohort.

### 5.3 Cohort Overview Page (/cohorts — new nav item)

A dedicated page showing:
- Grid of all cohorts, grouped by family
- Each card shows: name, donor count, total lifetime value, average score, trend arrow (growing/shrinking vs last period)
- Click into any cohort to see its members, insights, and recommended actions
- "Create Custom Cohort" button

### 5.4 Cohort Detail Page (/cohorts/[slug])

When you click into a cohort:
- Header: cohort name, description, member count, total value
- Score distribution chart (how are donors in this cohort scored?)
- Key stats: average gift size, average tenure, retention rate within cohort
- Member table (sortable, with all the same columns as lapsed scoring)
- "Generate Outreach for This Cohort" button → goes to AI Outreach with this cohort pre-selected
- "Export Cohort" button → downloads CSV of cohort members

### 5.5 CSV Upload Enhancement

After column detection (step 3 above), show a new step in the upload flow:
- "We detected these columns as potential donor segments:"
- List detected columns with checkboxes and sample values
- User checks which ones to import as cohorts
- Each selected column's unique values become individual cohort definitions under the ENGAGEMENT family

### 5.6 Dashboard Cohort Widget

On the main dashboard, add a "Cohort Snapshot" card:
- Shows top 5 cohorts by size or by urgency (e.g., "43 lapsed major donors", "12 churn-risk sustainers")
- Each row is clickable → navigates to that cohort's detail page

### 5.7 Outreach Cohort Context

When generating outreach in the AI Outreach Studio:
- The donor selection screen shows cohort badges next to each donor
- Filter donors by cohort before generating
- The AI prompt includes cohort context: "This donor is in the following cohorts: Major Donor, Gala Attendee, Lapsed. Tailor the outreach accordingly."
